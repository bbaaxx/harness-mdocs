## Summary

Jest coverage gates now require all global metrics to stay at or above 80%.

## Threshold

```js
coverageThreshold: {
  global: {
    statements: 80,
    branches: 80,
    functions: 80,
    lines: 80
  }
}
```

## Verified baseline

`npm run quality` passed with:

- Statements: 92.71%
- Branches: 80.83%
- Functions: 96.98%
- Lines: 95.67%

## Rationale

Branch coverage was the limiting metric. Once it exceeded 80%, statements/functions/lines had enough headroom to gate all metrics without making local development brittle.
