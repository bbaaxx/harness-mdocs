## Summary

Local quality tooling for `harness-mdocs` should stay simple and dependency-light.

## Commands

- `npm run lint` — TypeScript no-emit typecheck through `npm run typecheck`.
- `npm test` — full Jest suite.
- `npm run coverage` — Jest coverage report with text and lcov output collected from `src/**/*.ts`.
- `npm run mdocs:lint` / `npm run mdocs:validate` — build CLI and validate mdocs initiative/wiki/graph health.
- `npm run quality` — local aggregate: lint, tests, coverage, mdocs validation.
- `npm run release:check` — local quality plus `npm pack --dry-run`; this runs npm lifecycle hooks and may refresh generated plugin assets.

## Decisions

- Use TypeScript as initial lint/typecheck layer to avoid extra dependencies.
- Defer complexity analysis until a dedicated lint layer such as ESLint is justified.
- Keep GitHub Actions/CI integration out of this tooling initiative until surface tests are ready.
