# wiki.ingest surface coverage + byte-stable index contract (G2d)

`wiki.ingest` (G2) is reachable from every harness surface and never clobbers an externally-owned index.

## Surface coverage
| Surface | How `wiki.ingest` is reached |
|---|---|
| Claude Code (MCP) | dedicated `mdocs_ingest` tool |
| Claude Code (CLI) | `mdocs command wiki.ingest --json '{...}'` |
| OpenCode | dedicated `mdocs_ingest` tool (added G2d) AND the generic `mdocs` command tool |
| Codex | CLI (`commandTools:false`; `mdocs command wiki.ingest ...`) |

The generic command dispatcher (`core.commands.execute`) is the common spine; dedicated tools are convenience wrappers.

## Consumer skills
Every `mdocs-workflow` + `mdocs-orchestrator` SKILL.md copy now instructs using `wiki.ingest`/`mdocs_ingest` at REPORT to batch-emit wiki pages + overview/log sections from caller-supplied ops (no auto-prose).

## Byte-stable index contract (the correctness proof)
Directory-v2 (`canonical-lowercase`) `wiki/index.md` is owned externally by default. `tests/core/wiki-ingest-dir-v2.test.ts` proves:
- **External owner:** after `wiki.ingest` (createPage + updateOverviewSection + appendLog), `wiki/index.md` bytes are IDENTICAL (ingest routes through `wiki.create`/helpers that no-op the index when `wikiIndexOwner !== 'harness'`). overview.md/log.md ARE produced (they are harness-owned compiled views, distinct from the index).
- **Harness owner** (`wikiIndexOwner:'harness'` opt-in): `wiki/index.md` regenerates to include the new page + compiled views, still canonical-lowercase, pre-existing entries preserved.
- **No-prose:** created page body == caller content exactly; overview section body == caller body exactly; log block == caller content exactly.
