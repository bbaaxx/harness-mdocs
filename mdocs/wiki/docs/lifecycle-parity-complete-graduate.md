# Lifecycle parity: complete status, expectedDuration, graduate (G4, 0.5.0)

G4 brought directory-v2 lifecycle to parity with the mature local process: a distinct `complete` state, expected-duration signaling, graduation into compiled views, and three advisory lint rules.

## `complete` status parity
`complete` is now a distinct surfaced `Status` (directory-v2). Dir-v2 `markDone` writes `status: complete` on disk; it surfaces as `complete` (previously collapsed to `done`). `done` remains the flat-v1 alias. Both mean "completed"; `isCompleted(status)` (in `src/core/types.ts`) treats them equally and is used at every completed-check site: archive, `findBlocked`, `findOverdue`, the done-needs-stable-learning lint, and the registry archive guard. `normalizeInitiativeStatus` now maps `complete|completed`→`complete`, `done`→`done`.

## expectedDuration
Optional initiative field `'normal' | 'long' | 'suppress'` (frontmatter `expected_duration`). Drives `long-running-active`: normal warns if active > 14 days, long > 60, suppress never. Round-trips through flat + directory stores.

## lifecycle.graduate
Command `{ id, sections?: [{section, body}], logEntry? }`. Records a completed initiative's learning into `wiki/overview.md` (named H2 sections) + `wiki/log.md` (append entry) via the G2a helpers, under `withLock`, and stamps `initiative.graduated` (clears `graduation-due`). Rejects non-completed initiatives. Overview/log produced only in directory-v2; the `graduated` stamp applies in any mode. **No auto-prose** — only caller-supplied text. Added to `directoryNativeWrites` so it works in directory mode. Reached via `mdocs`/CLI/OpenCode generic tool.

## Lint rules (advisory warnings, zero score impact)
- **long-running-active** — active past its expected-duration window.
- **stale-complete** — completed > 30 days, not archived.
- **graduation-due** — completed > 7 days, not graduated.
