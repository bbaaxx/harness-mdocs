# wiki.ingest + compiled-knowledge model (G2, 0.5.0)

G2 closed the largest 0.5.0 gap: a `wiki.ingest` command that compiles initiative artifacts into curated wiki pages plus harness-owned `overview.md`/`log.md` views — without auto-generating prose. Split into four sub-initiatives, all shipped + verified.

## What shipped
- **G2a** — `overview.md` (section-addressable dashboard) + `log.md` (append-only) grammar and `WikiManager.updateOverviewSection()`/`appendLog()` helpers. Directory-v2 only; never touches the externally-owned `index.md`. See [[compiled-knowledge-overview-log-grammar]].
- **G2b** — `wiki.ingest` command (registry) composing `wiki.*` + the G2a helpers atomically under `withLock`; MCP `mdocs_ingest`; CLI help; `withLock` promoted to `src/core/lock.ts`. No auto-prose. See [[wiki-ingest-command]].
- **G2c** — `repos`/`systems` entity stub templates with `wiki.stub` auto-selection (prose-free). See [[repo-system-entity-templates]].
- **G2d** — surface coverage (OpenCode dedicated tool; Codex via CLI), skill guidance at REPORT, and the byte-stable `index.md` fixture/contract. See [[wiki-ingest-surface-coverage]].

## Core invariants
1. **No auto-prose.** The harness writes only structure; the agent authors all text. `wiki.ingest` records + applies caller-supplied operations only.
2. **Directory-v2 compiled views.** `overview.md`/`log.md` are harness-owned; produced only in `canonical-lowercase` mode.
3. **External-owner safety.** `wiki/index.md` stays byte-stable under the default external owner; regenerates only under the `harness` opt-in.
