# repos/systems entity stub templates (G2c)

Directory-v2 wiki layouts get two first-class entity categories (0.5.0): `repos` and `systems`. `WikiManager.stub()` auto-selects a dedicated prose-free template for these; all other categories keep the generic template. `wiki.create()` is unchanged (content is caller-supplied).

## Templates
- `repos/<id>.md` — frontmatter `category: "repos"`; sections: `## Summary`, `## Responsibilities`, `## Dependencies`, `## Owners / Links`.
- `systems/<id>.md` — frontmatter `category: "systems"`; sections: `## Summary`, `## Boundaries`, `## Dependencies`, `## Owners / Links`.

Both are **prose-free**: headings + `<!-- placeholder -->` comments only. The agent fills them in. Match `defaultStubTemplate`'s frontmatter format.

## Dispatch
`stub(category, id, title?, template?)` routes the default through `entityOrGenericStubTemplate(title, category, id, date)`: `repos` → repo template, `systems` → system template, else `defaultStubTemplate`. An explicit `template` argument always overrides (entity selection is only the default when no template is passed). Applied in both the root-category and category branches of `stub()`.

## Usage with ingest
`wiki.ingest` (G2b) `createPage` writes caller content verbatim; pair it with `wiki.stub` to scaffold an entity page, then `updatePage` to fill it.
