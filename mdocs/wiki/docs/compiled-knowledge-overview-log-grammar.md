# Compiled-knowledge overview.md / log.md grammar

Directory-v2 (`canonical-lowercase`) wiki layouts get two **harness-owned compiled views** maintained by `WikiManager` (G2a, 0.5.0). Both are produced ONLY in canonical-lowercase mode and NEVER write the externally-owned `wiki/index.md`.

## overview.md - section-addressable dashboard

`<mdocsRoot>/wiki/overview.md`. Minimal valid frontmatter (`id: "overview"`, `title: "Overview"`, `category: ""`, dates, empty arrays) so it passes `wiki.validate()` and appears in the canonical index. Body: one H1 `# Overview`, then named H2 sections `## <Section Name>`.

- `WikiManager.updateOverviewSection(section, body)` sets one section: **idempotent** - replaces the named section's body in place, preserving every other section byte-for-byte; appends a new section at the end if absent; bumps frontmatter `updated`.
- Sections keyed by exact heading text; a section body runs from the line after the heading to the next `^## ` line.

## log.md - append-only timeline

`<mdocsRoot>/wiki/log.md`. Same frontmatter shape (`id: "log"`, `title: "Log"`). Body: H1 `# Log`, then timestamped H2 blocks `## <iso-timestamp>` each with caller-supplied content.

- `WikiManager.appendLog(entry)` appends one block at the END (chronological), never reorders/rewrites existing entries. `entry` = `{ timestamp?, content }` or plain string; timestamp defaults to `new Date().toISOString()`.

## Invariants (enforced)

1. **No auto-prose.** Harness writes only structure (frontmatter, H1, H2 headers). All descriptive text is caller-supplied; the agent authors prose. `wiki.ingest` (G2b) records caller operations, never synthesizes.
2. **Directory-v2 only.** Both helpers `return null` and write nothing when `wikiIndexMode !== 'canonical-lowercase'`.
3. **index.md untouched.** Neither helper writes `wiki/index.md` directly. They call `updateIndices()` (no-op when `wikiIndexOwner !== 'harness'`): under the dir-v2 default owner (`external`), `index.md` stays byte-stable; under `harness` owner it regenerates to list the new views.

## Consumers

- `wiki.ingest` (G2b) composes these helpers atomically under `withLock`.
- `lifecycle.graduate` (G4) moves overview sections + appends a log entry on graduation.

Foundation for G2b, G2c, G4.
