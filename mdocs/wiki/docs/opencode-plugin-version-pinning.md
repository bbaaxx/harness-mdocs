# OpenCode plugin version pinning

For project configs, prefer pinned npm plugin specs such as `harness-mdocs@0.4.0` instead of `harness-mdocs@latest`.

Reason: OpenCode installs npm plugins into `~/.cache/opencode/packages/` at startup. Observed behavior: a previous `harness-mdocs@latest` cache remained pinned internally to an older package version after restart, while an exact spec (`harness-mdocs@0.4.0`) installed into a separate cache directory with the correct package version.

Migration guidance:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": ["harness-mdocs@0.4.0"]
}
```

If users previously used `@latest` or `opencode-mdocs`, remove stale caches before restarting:

```bash
rm -rf ~/.cache/opencode/packages/harness-mdocs@latest
rm -rf ~/.cache/opencode/packages/opencode-mdocs@*
```

Verify cache after restart:

```bash
node -p "require(process.env.HOME + '/.cache/opencode/packages/harness-mdocs@0.4.0/node_modules/harness-mdocs/package.json').version"
```

Expected output: `0.4.0`.
