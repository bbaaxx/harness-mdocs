## Summary

Evaluation and implementation learning for initiative `implement-directory-v2-compatibility`.

Goal: make harness-mdocs support both flat-v1 and directory-v2 mdocs contracts without clobbering canonical lowercase wiki indexes or rewriting directory initiatives into flat files.

## Stable outcome

Harness-mdocs is now contract-aware for flat-v1 and directory-v2 projects.

Implemented directory-v2 support includes:

- Contract detection/config override via `src/core/contract.ts`.
- Safe index ownership: lowercase `mdocs/wiki/index.md` is external-owned and never overwritten; uppercase `INDEX.md` generation is disabled for directory-v2 external indexes.
- Directory initiative reads from `initiatives/<slug>/_status.md`, including `_archive/`, legacy flat coexistence, status aliases, lookup/search/dispatch/validation routing.
- Native directory-v2 initiative writes for create, update, done, delete, and archive without creating flat initiative files.
- Root wiki pages as first-class entries for read, search, validation, create, update, delete, and stub.
- `wiki.link` for directory-v2 initiatives using native `_status.md` updates and wiki backlinks; accepts both `category/id` and root `id` references.
- Provenance validation through `source_initiatives` and legacy `sources` for stable wiki learning.
- Optional Obsidian visibility detection only; `_obsidian/` is ignored as canonical data and refresh commands are never auto-executed.

## Hotspots

- `src/core/managers/mdocs.ts`: init/exists previously assumed uppercase `wiki/INDEX.md`.
- `src/core/managers/initiative.ts`: flat initiative CRUD, parsing, archive, index generation.
- `src/core/initiative-store.ts`: directory-v2 `_status.md` read/write adapter.
- `src/core/managers/wiki.ts`: category wiki pages, root pages, generated index policy, backlinks.
- `src/core/search.ts`: initiative/wiki indexing across flat and directory contracts.
- `src/core/validation/linter.ts`: graph validation, stable learning gates, provenance.
- `src/core/operations.ts`: lookup/status/resume/dispatch/index checks.
- `src/core/commands/registry.ts`: command routing and write semantics.
- `src/surfaces/opencode/tools.ts`: surface-level parity.

## Guardrails

- Never treat `mdocs/_obsidian/` as canonical data.
- Never auto-run `obsidianRefreshCommand`; Obsidian integration remains opt-in metadata/visibility only.
- Never clobber lowercase canonical `wiki/index.md`; reject root wiki writes to `index`/`INDEX` in canonical-lowercase mode.
- Reject malformed wiki refs before writing initiative backlinks to avoid partial side effects.
- Normalize `complete`/`completed` to internal `done`, and `archive`/`archived` to `archived`.

## Regression coverage

- Directory-v2 fixture proves no unwanted uppercase index generation.
- Lowercase `wiki/index.md` remains byte-for-byte unchanged through validate/index sync/wiki operations.
- Directory initiatives are read and written without migration to flat files.
- Directory-v2 create/update/done/delete/archive preserve folder semantics.
- Root wiki create/update/delete/stub works without clobbering canonical index.
- Directory-v2 `wiki.link` supports category and root refs, normalizes `.md` suffixes, and rejects empty path segments.
- `_obsidian/` is ignored by list/search/validation.
