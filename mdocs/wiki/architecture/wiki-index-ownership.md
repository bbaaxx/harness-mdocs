# Wiki index ownership (directory-v2 opt-in)

In directory-v2 projects the canonical wiki index is the lowercase `wiki/index.md`. By default harness-mdocs treats it as **externally owned** (`wikiIndexOwner: 'external'`) and never touches it on `index.sync` — the safe default. G5 adds an opt-in for harness to maintain it.

## Opt-in
Set the contract to `wikiIndexMode: 'canonical-lowercase'` + `wikiIndexOwner: 'harness'`. Then `index.sync` regenerates `wiki/index.md` in the grouped/status-tagged canonical format.

## Owner/mode validation
`validateWikiIndexOwner` (src/core/contract.ts):
- `'harness'` — permitted for `generated-uppercase` (default) AND `canonical-lowercase` (opt-in).
- `'external'` — only meaningful for `canonical-lowercase` (dir-v2). External = harness no-ops on sync.
- `'none'` — no index.

## Sync gating
Every index path in `src/core/managers/wiki.ts` checks `wikiIndexOwner !== 'harness'` first and no-ops. So the default (external) is byte-stable: `index.sync` leaves `wiki/index.md` untouched.

## Non-breaking
Default owner external => byte-stable no-op (0.4.2 behavior). Existing dir-v2 installs unchanged.

## Scope note
This is WIKI index ownership only. The initiatives `INDEX.md` drift (separate data-hygiene) is unrelated.
