# SessionStart orientation hook

The Claude Code plugin registers a **SessionStart** hook that injects a compact mdocs orientation banner at the start of every fresh / resumed / cleared / compacted session, so the agent lands oriented without calling a tool first.

## Stdout contract
The hook writes JSON to stdout (only processed on exit 0):

```json
{"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":"<markdown string>"}}
```

`additionalContext` is the string Claude Code injects into the session before the first prompt. Capped at 10,000 chars by Claude Code, so the banner is a pointer, not a dump. Verified against https://code.claude.com/docs/en/hooks.

## Fail-open
ANY error -> exit 0 with no stdout. A parser/translator bug must never wedge session start. The banner is best-effort.

## Banner contents (compact)
- Initiative counts by status (active/done/archived + any other), total, current workflow step.
- Active initiative id + title (if any).
- Wiki page count.
- Pointer: `mdocs_status` / `mdocs/wiki/index.md`.

## Registration
`src/surfaces/claude-code/plugin/.claude-plugin/plugin.json` and `assets/templates/settings-patch.json` register:
- `SessionStart` matcher `startup|resume|clear|compact` -> `dist/cli/hooks/session-start.js`
- `PreCompact` matcher `*` -> `dist/cli/hooks/pre-compact.js` (re-emits so orientation survives compaction)

## Project root
Uses the shared `resolveProjectRoot` (see [[project-root-resolution]]) so the orientation hook, the MCP server, and the PreToolUse/PostToolUse hooks all agree on the same mdocs root.

## Composing with consumer SessionStart hooks
Consumers can add their own SessionStart hooks; Claude Code runs all matching SessionStart hooks and concatenates their `additionalContext`. Keep each hook's output compact so the combined context stays useful. (See consumer-layering docs, G8.)
