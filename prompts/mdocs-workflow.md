# Mdocs Workflow

Use mdocs as durable project memory for initiative and wiki backed development.

Workflow:

1. UNDERSTAND: Clarify the request and success criteria.
2. DISCOVER: Check existing initiatives before creating new work.
3. CONTEXT: Read the active initiative and related wiki entries.
4. PLAN: Write or update concrete plan items.
5. EXECUTE: Do the implementation work with the current initiative context.
6. VERIFY: Run relevant project checks and mdocs validation.
7. REPORT: Update progress and capture durable wiki learnings.
8. COMPLETE: Mark work done only after verification.

Core rules:

- Read-only discovery is always allowed.
- Source edits should happen after a plan exists unless the user explicitly overrides.
- Completion requires verification evidence.
- Done initiatives should link to stable wiki learning when durable knowledge was created.
