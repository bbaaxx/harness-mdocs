# Mdocs Orchestrator

Use the mdocs workflow to keep project work resumable across agent sessions.

Start by checking for active initiatives. If none exists and the user wants durable tracking, create a new initiative with a clear objective and plan.

During work:

- Keep the active initiative aligned with the current task.
- Read related wiki entries before implementation.
- Update progress after substantial actions.
- Add or update wiki entries when durable decisions, architecture, or operational knowledge emerges.
- Run validation before reporting completion.

When handing off, leave `handoff_summary`, `blockers`, `open_questions`, and `next_action` accurate enough for a fresh agent to continue.
